---
title: 'Reset admin password'
description: 'Recover access to the Logwiz admin account by deleting the admin user and completing the setup wizard.'
---

If you lose access to the admin account, you can delete the admin user from the database and let Logwiz guide you through creating a new one via the setup wizard.

<Warning>
	This permanently deletes the admin user record, including all associated sessions and linked
	accounts. The cascade is handled automatically via foreign-key relations in the database.
</Warning>

1. Stop the server:

   ```bash
   docker compose stop logwiz
   ```

2. Open the SQLite database and delete the admin row:

   ```bash
   sqlite3 ./data/logwiz.db "DELETE FROM user WHERE role = 'admin';"
   ```

   If you are running inside Docker, exec into the container first:

   ```bash
   docker exec -it logwiz sqlite3 data/logwiz.db "DELETE FROM user WHERE role = 'admin';"
   ```

3. Restart the server:

   ```bash
   docker compose start logwiz
   ```

4. Open [http://localhost:8282](http://localhost:8282) in your browser. Because there is no admin account, Logwiz automatically redirects you to `/auth/setup-admin`. Enter your desired admin credentials and submit the form.

<Note>
	After completing the wizard, the new admin account is active immediately. No restart is required.
</Note>
