---
title: 'Reset admin password'
description: 'Recover access to the Logwiz admin account by deleting every admin user and completing the setup wizard.'
---

If you lose access to the admin account, delete every admin row from the database and let Logwiz guide you through creating a new one via the setup wizard. The setup wizard is only served while zero admin accounts exist, so any remaining admin account must also be removed.

<Warning>
	This permanently deletes every admin user record, including all associated sessions and linked
	accounts. The cascade is handled automatically via foreign-key relations in the database. If you
	only want to replace a single admin, sign in with another admin and use **Administration → Users →
	Reset password** instead.
</Warning>

### Docker Compose (default install)

The bundled compose file stores the database on a named volume, so there is no host `./data`
directory to point `sqlite3` at. Run the delete inside the running container and then restart it:

```bash
docker exec -it logwiz sqlite3 /app/data/logwiz.db "DELETE FROM user WHERE role = 'admin';"
docker compose restart logwiz
```

### Bare-metal install

If `LOGWIZ_DATABASE_PATH` is a host path you control, stop the server, edit the database, and
start it again:

```bash
# Adjust the path to match LOGWIZ_DATABASE_PATH
sqlite3 ./data/logwiz.db "DELETE FROM user WHERE role = 'admin';"
```

### Finish in the UI

Open [http://localhost:8282](http://localhost:8282) in your browser. Because there is no admin account, Logwiz redirects you to `/auth/setup-admin`. Enter your desired admin credentials and submit the form.

<Note>
	After completing the wizard, the new admin account is active immediately. No restart is required.
</Note>
